### The problem I kept running into

Wanting to catch pink trumpet trees in bloom, or spot a Purple Sunbird in Bangalore — but the only options were:
- Scroll Instagram hoping someone posted the right location
- Ask random people on birding forums and wait
- Drive out and recce the place yourself

So I built tools to fix that.

---

### What I've built

#### 🌸 [Pink Trees Bangalore](https://github.com/ritwikraj16/pink-trees-bangalore) · [Live →](https://pink-trees-bangalore.vercel.app)
Real-time map of pink-flowering trees across Bangalore. Pulls from satellite imagery (Sentinel-2), iNaturalist community sightings, OpenStreetMap, and BBMP's tree census — deduplicated and served on a live map.
> *No more guessing which road is blooming this week.*

#### 🐦 [Bangalore Bird Sightings](https://github.com/ritwikraj16/bangalore-bird-sightings)
Month-wise sighting probability for bird species across Bangalore's hotspots — Cubbon Park, Lalbagh, Bannerghatta and more. Powered by iNaturalist's historical research-grade observations.
> *Know which species to expect, at which area, in which month — before you leave the house.*

#### ✈️ [Realtime Flight Tracker](https://github.com/ritwikraj16/realtime-flight-tracker)
Live flight tracking dashboard built with TypeScript.

### How I think about this

I'm a PM by day. These projects aren't just side hacks — each one starts with a real frustration, maps to a data source that already exists but isn't surfaced well, and ends with something usable by a non-technical person.

The stack is secondary. The question I always start with: *what does someone actually need to know, and where does that data already live?*
